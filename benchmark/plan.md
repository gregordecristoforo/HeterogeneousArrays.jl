# Rewrite the benchmark to include more combinations of array interfaces and unit handling libraries

The script `runbenchmark.jl` should be rewritten in order to include combinations of array interface (ComponentVector, ArrayPartition, HeterogeneousVector, and FieldVector), unit handling (None (plain floats), Unitful, and FlexUnits), and ODE function interface (allocating (the ODE function returns an array), and non-allocating (the ODE function modifies an array in place)). Some combinations are not compatible and should hence not be tested: These are the combinations of FlexUnits with the non-allocating interface and FieldVector with non-allocating interface. The running numbers in the table before the description of the method should be omitted. Make separate columns in the table: Array structure, Unit handling, and ODE interface.
